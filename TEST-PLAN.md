# Mission Control Next - 测试计划 (Test Plan)

## 1. 测试策略

### 1.1 测试目标
确保 Mission Control Next 项目的功能完整性、性能稳定性和安全性，为生产环境部署做好准备。

### 1.2 测试范围
- **Tasks Board 模块**: 任务看板、拖拽功能、CRUD 操作
- **Calendar 模块**: Cron Job 管理、定时任务调度
- **Memory 模块**: 记忆存储、搜索、标签过滤
- **Agent 同步模块**: 状态上报、心跳机制
- **基础架构**: Next.js 构建、TypeScript 类型检查、Convex API

### 1.3 测试方法
| 测试类型 | 工具/方法 | 频率 |
|----------|----------|------|
| 单元测试 | Vitest / Jest | 每次代码变更 |
| 集成测试 | Convex Test Helpers | 每次代码变更 |
| E2E 测试 | Playwright | 每周/发布前 |
| 性能测试 | Lighthouse / WebPageTest | 发布前 |
| 安全审计 | Gemini CLI / 手动审查 | 发布前 |

## 3. 测试环境配置

### 3.1 开发环境
```bash
# Node.js 版本
node v22.x

# 依赖安装
cd mission-control-next
npm install

# 环境变量配置 (.env.local)
NEXT_PUBLIC_CONVEX_URL=https://your-project.convex.cloud
```

### 3.2 测试命令
```bash
# TypeScript 类型检查
npx tsc --noEmit

# 生产构建测试
npm run build

# 数据导入脚本测试 (Dry Run)
npx tsx scripts/import-data.ts --memories --dry-run

# Agent 心跳脚本测试
npx tsx scripts/agent-heartbeat.ts

# Convex 函数测试
npx convex run "tasks.js:list"
npx convex run "agents.js:list"
npx convex run "memories.js:list"
```

### 3.3 测试数据准备
- 使用 `scripts/import-data.ts` 导入测试记忆数据
- 手动创建测试任务和 Agent 记录
- 准备边界测试数据（超长文本、特殊字符等）

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Convex 未部署 | 无法进行集成测试 | 使用 Mock Client 进行离线测试 |
| 测试数据不足 | 测试覆盖不全 | 使用 import-data.ts 导入真实数据 |
| 时间不足 | 测试不完整 | 优先执行 P0 测试用例 |
| 环境问题 | 测试无法执行 | 准备 Docker 容器化测试环境 |
